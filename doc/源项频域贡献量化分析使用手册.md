# 源项频域贡献量化分析使用手册

## 📖 文档概述
本工具是旋翼气动噪声分析系统的扩展功能，实现了噪声源的精细化频域贡献量化分析，能够分离谐频/宽频噪声，分解厚度噪声、定常载荷噪声、非定常载荷噪声的能量占比，为噪声机理分析和降噪设计提供数据支撑。

## 🧮 核心算法原理

### 1. 谐频与宽频噪声分离（谐频提取法）
**物理依据**：旋翼气动噪声的谐频分量集中在叶片通过频率及其整数倍频率处，呈现离散峰值特征；宽频分量则是连续分布的背景噪声。

**算法流程**：
1. 识别基频（自动识别或手动指定）
2. 生成各阶谐频的理论频率（基频 × 阶次）
3. 对每个谐频取±3%（可配置）的带宽范围
4. 将带宽内的频谱能量归为谐频分量
5. 剩余频谱能量归为宽频分量
6. 验证能量守恒：谐频能量 + 宽频能量 ≈ 原始总能量（误差≤1%为正常）

### 2. 定常与非定常载荷噪声分离（相位约束法）
**物理依据**：
- 定常载荷噪声：由定常气动力周期性作用产生，是确定性周期信号。以厚度噪声相位为基准，在基准轴上的同相投影为定常分量
- 非定常载荷噪声：由随机因素产生，相位与厚度噪声无固定关系。在基准轴上的正交投影为非定常分量

**算法流程**：
1. 识别所有谐频的带宽范围
2. 提取厚度噪声在各谐频带的平均相位作为基准相位
3. 计算载荷噪声在各谐频处相对于基准相位的相位差
4. 同相投影（cos 分量）→ 定常载荷噪声；正交投影（sin 分量）→ 非定常载荷噪声
5. 谐频带外的宽频区域全部归为非定常载荷
6. 验证能量守恒：|定常|² + |非定常|² = |原始载荷|²（数学上严格守恒，误差 ≤ 0.1%）

> **注意**：相位约束法的分离正确性依赖一个隐含假设——定常载荷与厚度噪声在观测者位置"同相"。这一假设在物理上不一定成立，详见技术方案文档。如果分离结果不符合物理预期（如 BPF 处非定常占比异常高），建议使用循环平稳谱分析法进行交叉验证。

### 3. 频带贡献量化分析
**实现功能**：
- 支持1/1、1/3、1/12倍频程划分（默认1/3倍频程）
- 支持自定义频带划分
- 自动计算每个频带内：
  • 厚度噪声、定常载荷、非定常载荷的能量占比
  • 谐频与宽频的能量占比
  • 各噪声源的等效声压级

## 📊 输出文件详解
所有分析结果保存为CSV格式，方便后续处理和绘图。

### 文件1：`*_SourceContribution_Detail.csv`（频率点级详细数据）
**用途**：精细化频谱分解，查看每个频率点的噪声源组成
| 列名 | 含义 |
|------|------|
| `Frequency(Hz)` | 频率值 |
| `[源名称]_amp(Pa)` | 该频率点对应噪声源的线性幅值 |
| `[源名称]_SPL(dB)` | 该频率点对应噪声源的声压级 |
| `[源名称]_energy` | 该频率点对应噪声源的能量（幅值平方） |

**源名称说明**：
- `thickness`：厚度噪声
- `steady_load`：定常载荷噪声
- `unsteady_load`：非定常载荷噪声
- `total`：总噪声（厚度 + 载荷）
- `harmonic`：所有谐频成分总和
- `broadband`：所有宽频成分总和
- （含表面反射场景会增加`FF_`/`SR_`/`merged_`前缀，分别对应自由场、表面反射、合成信号）

---

### 文件2：`*_HarmonicContribution.csv`（谐频贡献分析）
**用途**：分析各阶谐频的噪声源组成
| 列名 | 含义 |
|------|------|
| `Harmonic Order` | 谐频阶次（1=基频，2=2倍频，...） |
| `Nominal Frequency(Hz)` | 理论谐频频率（基频 × 阶次） |
| `Actual Frequency(Hz)` | 频谱中实际最接近的频率点 |
| `[源名称]_amp(Pa)` | 该谐频点对应噪声源的幅值 |
| `[源名称]_SPL(dB)` | 该谐频点对应噪声源的声压级 |
| `[源名称]_ratio` | 该谐频点对应噪声源的能量占比（相对于该点总能量） |

---

### 文件3：`*_BandSourceContribution.csv`（频带贡献分析）
**用途**：1/3倍频程频带的噪声源分布和谐频占比分析
| 列名 | 含义 |
|------|------|
| `Center Frequency(Hz)` | 频带中心频率 |
| `Lower Bound(Hz)` | 频带下边界 |
| `Upper Bound(Hz)` | 频带上边界 |
| `[源名称]_energy` | 该频带内对应噪声源的总能量 |
| `[源名称]_ratio` | 该频带内对应噪声源的能量占比（相对于该频带总能量） |
| `[源名称]_spl` | 该频带内对应噪声源的等效声压级 |
| `harmonic_ratio_in_band` | ✅ 核心指标：该频带内谐频能量占总能量的比例 |

---

### 文件4：`*_SourceContribution_Summary.csv`（全局汇总统计）
**用途**：全频段的噪声源贡献总览
| 列名 | 含义 |
|------|------|
| `Filename` | 测点文件名 |
| `Fundamental Frequency(Hz)` | 基频（自动识别或手动指定） |
| `Number of Harmonics` | 识别到的谐频总数 |
| `[源名称]_total_energy` | 全频段对应噪声源的总能量 |
| `[源名称]_total_ratio` | ✅ 核心指标：全频段对应噪声源的能量占比 |
| `[源名称]_total_spl` | 全频段对应噪声源的总声压级 |
| `[源名称]_low_freq_ratio` | 低频段（<250Hz）对应噪声源的能量占比 |
| `[源名称]_mid_freq_ratio` | 中频段（250~2000Hz）对应噪声源的能量占比 |
| `[源名称]_high_freq_ratio` | 高频段（≥2000Hz）对应噪声源的能量占比 |
| `harmonic_total_ratio` | ✅ 核心指标：全频段谐频能量占**所有噪声源总能量**的比例（谐频能量 / 总噪声能量） |
| `harmonic_to_broadband_ratio` | ✅ 核心指标：全频段谐频能量占**（谐频+宽频）总能量**的比例（谐频能量 / (谐频能量+宽频能量)） |

## 🚀 使用指南

### 方式1：命令行使用（推荐）
通过`main.py`统一入口，支持四种分析模式：

#### 查看帮助
```bash
python main.py --help
python main.py source --help  # 查看源项分析的参数说明
```

#### 模式1：仅源项分析
```bash
# 仅自由场数据（对应analysis_cases01.py格式）
python main.py source 数据目录 文件名前缀1 文件名前缀2 --group-prefixes 分组前缀

# 含表面反射数据（对应analysis_cases.py格式）
python main.py source 数据目录 文件名前缀 --has-reflection --group-prefixes 分组前缀

# 手动指定基频（推荐，提高分析精度）
python main.py source 数据目录 文件名前缀 --fundamental-freq 25.0
```

#### 模式2：完整分析流程（峰值分析 + 频带分析 + 源项分析）
```bash
python main.py full 数据目录 文件名前缀 --has-reflection --fundamental-freq 25.0
```

#### 模式3：原有功能（峰值分析/频带分析）
```bash
# 峰值分析
python main.py peak 数据目录 文件名前缀

# 频带分析
python main.py band 数据目录 文件名前缀 --group-prefixes 分组前缀
```

#### 常用参数说明
| 参数 | 含义 | 默认值 |
|------|------|--------|
| `--has-reflection` | 数据是否包含表面反射分量（FF/SR/merged），如果你的数据是`*_FreqDomain.csv`里包含`amp_FF_Total`这类字段，就需要加这个参数 | 关闭 |
| `--fundamental-freq` | 手动指定基频（叶片通过频率，计算公式：`叶片数 × 转速(rpm) / 60`），比自动识别更准确 | 自动识别 |
| `--harmonic-bandwidth-ratio` | 谐频带宽比，相对于谐频频率的比例，如0.03表示±3%带宽 | 0.03 |
| `--max-harmonic-order` | 最大分析的谐频阶次 | 30 |
| `--band-fraction` | 倍频程分数：1=1/1倍频程，3=1/3倍频程，12=1/12倍频程 | 3 |
| `--group-prefixes` | 分组保存汇总文件的前缀，比如所有测点都以`Case01_Rotor_`开头，就可以指定`--group-prefixes Case01_Rotor`，会生成统一的汇总文件 | 不分组 |

## ❓ 常见问题解答

### Q1：运行时出现`Warning: Energy conservation error XX% in load separation`是什么意思？
A：这个警告表示定常/非定常载荷分离后的能量总和与原始载荷能量相差超过了 0.1% 的阈值。
**常见原因**：
- 基频自动识别不准确，导致谐频位置找错
- 谐频带宽设置过小，没有覆盖完整的谐频能量
- 数值精度问题（极少见，通常由浮点运算累积误差导致）

**解决方法**：
1. 优先**手动指定基频**（最有效），通过`--fundamental-freq`参数传入实际的叶片通过频率
2. 适当增大谐频带宽比，如`--harmonic-bandwidth-ratio 0.05`
3. 增加最大谐频阶数，如`--max-harmonic-order 50`

> **关于分离结果的物理合理性**：能量守恒通过不代表分离结果物理正确。如果定常/非定常的比例不符合物理预期（如 BPF 处非定常占比异常高），可能是因为载荷噪声与厚度噪声在观测者位置的相位差接近 ±90°，导致相位约束法的"同相假设"不成立。此时建议参考《旋翼噪声定常非定常载荷分离技术方案》中关于方法局限性的讨论。

---

### Q2：自动识别的基频不对怎么办？
A：直接通过`--fundamental-freq`参数手动指定即可，手动指定的优先级高于自动识别。
基频计算公式：`叶片数 × 旋翼转速(rpm) / 60`
例如：4片桨叶，转速600rpm，基频就是 `4 × 600 / 60 = 40 Hz`

---

### Q3：如何调整频带划分方式？
A：通过参数调整：
- 使用1/1倍频程：`--band-fraction 1`
- 使用1/12倍频程：`--band-fraction 12`
- 使用自定义频带：`--band-type custom`（默认分为7个频带：20-100、100-160、160-315、315-630、630-1250、1250-2500、2500-5000Hz）

---

### Q4：含表面反射的数据为什么要加`--has-reflection`参数？
A：因为两种数据格式的列名不同：
- 无反射数据的列名是`amp_Total`、`amp_Thickness`、`amp_Load`
- 有反射数据的列名是`amp_FF_Total`、`amp_SR_Thickness`、`amp_merged_Load`等，包含FF/SR/merged三个分量的前缀
加了`--has-reflection`参数后，程序会自动适配对应的数据格式，对三个分量分别进行分析。
